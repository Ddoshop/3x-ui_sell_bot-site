# VPN Vouchers Bot

Telegram бот для активации VPN подписок через ваучер-коды с красивым интерфейсом.

## Быстрый старт

```bash
npm install
cp .env.example .env
npm start
```

## Переменные окружения

```env
# Обязательные
TELEGRAM_BOT_TOKEN=xxx                   # Токен бота из @BotFather
PAYMENTS_API_URL=http://localhost:8788   # URL API платежей

# Опциональные
INTERNAL_API_TOKEN=
BOT_CHAT_ID=123456789
BRAND_NAME=VPN Premium
BRAND_EMOJI=🌐
WEBHOOK_URL=
WEBHOOK_PATH=/bot-webhook
WEBHOOK_PORT=8080
BOT_HEARTBEAT_INTERVAL_MS=60000
```

Примечание: при пустом `WEBHOOK_URL` бот работает в polling-режиме.

## Получение токена бота

1. Откройте Telegram
2. Найдите @BotFather
3. Отправьте `/newbot`
4. Следуйте инструкциям
5. Скопируйте токен в переменную окружения

```

  ├── index.js      # Основной файл с логикой бота
  ├── config.js     # Конфигурация
  ├── ui.js         # Интерфейс (тексты, клавиатуры)

      └── api.js    # Запросы к API
```

## Состояния бота

Бот отслеживает состояние каждого пользователя:

- `main` - главное меню
- `select_plan` - выбор тарифа
- `plan_selected_*` - просмотр платежей
- `waiting_voucher` - ожидание ввода ваучера

## Интеграция с API

Бот общается с API платежей на `http://localhost:8788`:

```
GET  /api/plans
POST /api/payments/create
POST /api/vouchers/activate
GET  /api/users/:userId/subscriptions
```

## Команды бота

- `/start` - Начать встречу
- `/plans` - Показать тарифы
- `/cancel` - Отменить текущее действие

## Примеры использования

### Активация ваучера в боте

1. Пользователь нажимает "🎟️ Ввести ваучер"
2. Вводит код: `XXXX-XXXX-XXXX-XXXX`
3. Бот отправляет на API
4. API активирует ваучер и создаёт доступ
5. Бот показывает подтверждение и сроки

### Выбор тарифа

1. Пользователь нажимает "📦 Выбрать тариф"
2. Видит список тарифов с ценами
3. Выбирает подходящий
4. Получает реквизиты платежа
5. После оплаты получает ваучер от администратора в отдельном сообщении

## Обработка ошибок

При любой ошибке бот:
- Показывает понятное сообщение об ошибке
- Возвращает пользователя в главное меню
- Логирует ошибку в консоль для отладки

## Логирование

Консоль показывает:
- Запуск/остановку бота
- Ошибки при запросах к API
- Действия пользователей (опционально)
